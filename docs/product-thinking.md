# Product Thinking

## Users

- **Central command administrators** need an overview of risk, staffing, and incidents.
- **Sector officers** need localized signals and dispatch instructions.
- **Transport managers** need queue and turnaround visibility.
- **Medical and safety teams** need early warnings before pressure becomes an emergency.

## Core Workflow

1. Watch the live overview for rising risk.
2. Inspect sector heatmap to locate the pressure point.
3. Review recommended actions.
4. Sort incident queue by SLA.
5. Redeploy resources.
6. Export a short brief for handover.

## Design Principles

- Show the next decision, not just the data.
- Keep language short enough for a noisy control room.
- Make scenario shifts visible in one click.
- Prefer boring reliability over fragile demos.
- Build the first version so it can run even when connectivity is imperfect.

## Differentiator

The dashboard is not just a collection of charts. It joins signals into operational guidance: where to move people, which team to stage, and what needs attention before the next crowd pulse.
